# 용어집 도구 (한↔영 게임 번역용)

게임 회사 통번역사를 위한 단일 파일 용어집 관리 도구.
설치 불필요 — `index.html`을 더블클릭하면 브라우저에서 바로 열립니다.

## 핵심 기능

1. **용어 등록/검색/관리** — 한국어, 영어, 예문, 프로젝트 태그
2. **엑셀 하이라이트 보기** — 작업 엑셀을 업로드하면 등록된 용어를 자동으로 색칠하고 영문 번역을 옆에 표시 (용어집을 "찾으러 가지 않아도" 됨)
3. **AI 후보 추출** — 한/영 컬럼이 있는 작업 파일을 업로드하면 Claude AI가 용어집에 추가할 만한 후보를 뽑아줌 (등록 마찰 최소화)
4. **JSON 백업/복원** — 데이터를 파일로 내보내거나 가져올 수 있음 (팀 공유, 다른 PC로 이전)

## 시작하기

### 1단계: 파일 열기

`index.html` 파일을 더블클릭하세요. 기본 브라우저(Chrome/Edge 권장)에서 열립니다.

> 📌 **Chrome/Edge가 권장됩니다.** Firefox/Safari도 동작하지만 일부 기능에서 차이가 있을 수 있어요.

### 2단계: 첫 용어 등록 (선택)

- 상단 **📚 용어집** 탭 → **+ 새 용어** 버튼
- 한국어, 영어 입력 → 저장
- 또는 아래 "AI 후보 추출"로 한꺼번에 등록 가능

### 3단계: Claude API 키 등록 (AI 기능을 쓰려면)

AI 후보 추출 기능을 사용하려면 Claude API 키가 필요합니다. **AI 기능을 안 써도 다른 기능은 모두 동작합니다.**

1. https://console.anthropic.com/ 가입 후 결제 정보 등록
2. API Keys 메뉴에서 키 발급 (sk-ant-... 로 시작)
3. 본 도구의 **⚙️ 설정** 탭 → API 키 입력 → 저장
4. 모델 선택:
   - **Opus 4.7** — 정확도 최고, 비용 가장 높음
   - **Sonnet 4.6** — 균형 (권장 시작점)
   - **Haiku 4.5** — 빠르고 저렴, 간단한 작업용

## 주요 사용 시나리오

### 시나리오 A: 새 프로젝트 시작 시 용어집 빠르게 구축

1. **⚙️ 설정** 탭 → 프로젝트 추가 (예: "신작 A")
2. **✨ AI 후보 추출** 탭 → 과거 한/영 번역 완료된 엑셀 업로드
3. 한국어 컬럼 / 영어 컬럼 / 프로젝트 선택 → AI 분석 시작
4. AI가 뽑아준 후보 검토 → 체크박스로 선택 → 용어집에 추가

### 시나리오 B: 작업 중 등록된 용어 확인

1. **🔍 하이라이트 보기** 탭 → 작업 중인 엑셀 업로드
2. 한국어 컬럼 선택 → 하이라이트 보기
3. 등록된 용어가 노란색으로 표시되고, 영문 번역이 옆에 작게 붙음
4. 페이지 상단에 "이 파일에서 발견된 용어" 목록도 표시

### 시나리오 C: 작업 중 새 용어 발견

1. **📚 용어집** 탭 → **+ 새 용어** → 한국어/영어/예문 입력
2. 또는 작업 끝나면 시나리오 A 방식으로 한꺼번에 추출

## 데이터는 어디에 저장되나요?

- 모든 데이터는 **브라우저의 localStorage**에 저장됩니다 (이 PC, 이 브라우저에만)
- 다른 PC로 옮기거나 팀과 공유하려면: **⚙️ 설정 → JSON으로 내보내기**
- 다른 PC에서 받은 JSON을 합치려면: **⚙️ 설정 → JSON 가져오기** (중복은 자동 제외)

> ⚠️ **브라우저 데이터를 지우면 용어집도 같이 사라집니다.** 정기적으로 JSON으로 내보내서 백업해두세요.

## 보안 / 프라이버시

- API 키와 용어 데이터는 본 PC의 브라우저에만 저장됩니다
- AI 후보 추출 시에만 Claude API(Anthropic 서버)로 텍스트가 전송됩니다
- 그 외에는 어떤 외부 서버와도 통신하지 않습니다
- 회사 보안 정책상 외부 API 사용이 제약된다면, AI 기능은 끄고 수동 등록 + 하이라이트 기능만 쓰셔도 됩니다 (이 두 기능은 외부 통신 없음)

## 팀 공유

### Phase 1 — 도구 자체 공유 (GitHub Pages)

도구를 GitHub Pages에 무료로 배포해서 팀원이 URL로 접속할 수 있게 만듭니다. 각 팀원은 본인 브라우저(localStorage)에 데이터를 따로 저장하므로, **용어집은 각자 관리**합니다. 필요할 때 JSON 내보내기/가져오기로 데이터를 주고받으면 됩니다.

**배포 단계 (1회만):**

1. github.com 가입 (무료)
2. 우측 상단 **+** → **New repository** 클릭
3. Repository name 입력 (예: `glossary-tool`) → **Public** 선택 → README 체크박스 끔 → **Create repository**
4. 빈 repo 페이지에서 **uploading an existing file** 링크 클릭
5. `index.html`, `app.js`, `README.md` 세 파일을 드래그앤드롭으로 업로드 → **Commit changes**
6. repo 상단의 **Settings** 탭 → 왼쪽 사이드바 **Pages**
7. Source 섹션에서 Branch를 `main` 으로 선택 → **Save**
8. 1~2분 후 페이지 상단에 활성화된 URL이 표시됨 (예: `https://본인계정.github.io/glossary-tool/`)
9. 그 URL을 팀원들에게 공유하면 끝

**팀원이 해야 할 것:**

- URL 접속 → **⚙️ 설정** 탭에서 본인 Claude API 키 등록 (AI 후보 추출 사용 시)
- 본인이 추가하는 용어는 본인 브라우저에만 저장됨
- 한 명이 마스터 용어집을 관리한다면, 그 사람이 주기적으로 JSON 파일을 내보내서 팀에 공유 → 팀원들이 "JSON 가져오기"로 합침

### Phase 2 — 용어집 자동 공유 (Supabase 백엔드, 추후 추가)

Phase 1 운영을 해보고 "용어집 자체도 실시간으로 공유됐으면 좋겠다"는 결론이 나면, Supabase 백엔드를 붙여서 확장합니다.

**예정 구조:**

- **GitHub Pages** — 도구 호스팅은 그대로 유지
- **Supabase** (무료) — PostgreSQL DB + 이메일 인증 + Realtime 동기화
- 팀원 각자 본인 이메일로 로그인 → 같은 워크스페이스의 공유 용어집을 보고 편집
- 한 명이 추가/수정하면 다른 팀원에게 1~2초 안에 반영
- 권한 분리도 가능 (예: 일부는 읽기 전용)

Phase 2를 추가할 때:

1. Supabase 프로젝트 생성 (무료, 5분)
2. 도구 코드에 Supabase 연동 추가
3. 기존 localStorage 데이터를 Supabase로 마이그레이션
4. 새 버전 GitHub에 push → 자동 배포

**보안 / NDA 검토 필요:** Supabase에 게임 관련 용어를 저장하는 것이 회사 정책상 가능한지 사전 확인 필요. 무료 티어 데이터는 Supabase 인프라(AWS 리전 선택 가능)에 저장됩니다.

## 문제 해결

**Q. AI 후보 추출이 실패해요.**
- API 키가 올바른지 확인 (sk-ant- 로 시작하는지)
- Anthropic 콘솔에서 크레딧이 남아있는지 확인
- 한 번에 너무 많은 행을 분석하면 실패할 수 있어요. "최대 분석 행 수"를 100 이하로 줄여보세요.

**Q. 엑셀 파일이 안 열려요.**
- .xlsx, .xls, .csv 만 지원합니다. 다른 형식이면 엑셀에서 다시 저장해서 시도하세요.
- 파일이 너무 크면 시간이 걸릴 수 있어요 (수천 행까지는 보통 즉시 처리).

**Q. 용어가 하이라이트 안 돼요.**
- 용어집의 한국어와 엑셀의 한국어가 **정확히 일치**해야 합니다 (띄어쓰기, 조사 포함). 부분 매칭이라 "길드"는 "길드원"에서도 잡히지만, "길  드"(공백 다름)는 안 잡혀요.

**Q. 데이터가 사라졌어요!**
- 브라우저 캐시/사이트 데이터 삭제 시 함께 사라집니다.
- 정기 백업 (JSON 내보내기)을 습관화하세요.
